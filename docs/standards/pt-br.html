Padrões

Padrões Compartilhados para Automação Prática de FinOps

Expectativas leves de engenharia para código reutilizável, sustentável, compreensível e seguro para rodar em diferentes ambientes.

English Version

Expectativas Centrais

  • Preferir contribuições simples e focadas em vez de abstrações pesadas.
  • Favorecer scripts standalone que resolvam um problema claro de FinOps.
  • Usar parâmetros e configuração em vez de detalhes hardcoded de ambiente.
  • Exigir testes para código submetido.
  • Manter o código legível, sustentável e pronto para open source.

Expectativas de Segurança

  • Adotar comportamento seguro por padrão.
  • Usar dry-run quando fizer sentido.
  • Evitar ações destrutivas a menos que sejam explicitamente habilitadas.
  • Fornecer logs ou saídas claras para consciência operacional.
  • Tornar o tratamento de erros compreensível e previsível.

Tratamento de Dados Sensíveis

Os repositórios não devem incluir:

  • IDs de conta
  • tokens ou segredos
  • endpoints privados
  • configuração interna restrita
  • informações confidenciais específicas de uma organização

Adequação Open Source

As contribuições devem ser escritas para reutilização fora do ambiente original. Isso significa documentar premissas, evitar contexto privado e tornar scripts fáceis de inspecionar e adaptar.
