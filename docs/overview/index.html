Overview

FinOps-as-Code Project Vision

FinOps-as-Code exists to turn FinOps decisions into practical automation. This initiative starts from a simple belief: visibility alone does not eliminate waste. Teams need simple, executable, and safe code they can use to act on cost, efficiency, and governance opportunities.

Portuguese Version

Current Focus

The current direction is to publish automation that solves real cloud financial operations problems with clear execution paths. Today that shows up mostly as scripts and lean workflows, because that format is the fastest way to prove usefulness, adoption, and reuse.

What The Initiative Publishes Today

  • cleanup_block_storage_idle targets unused block storage resources that continue to generate avoidable cloud cost.
  • cleanup_ip_idle focuses on reclaiming idle IP-related resources that create waste or weak governance.
  • linktree-f2-brazil supports the public visibility layer of the organization ecosystem.

What Fits The Initiative

A strong project for FinOps-as-Code should solve a concrete cost, efficiency, or governance problem, be easy to understand, safe to evaluate, and realistic for other teams to adapt. The goal is not to limit the work to the current repositories. The goal is to keep a clear bar for what is worth publishing under this official initiative.

How The Initiative Can Grow

New repositories can be created whenever a legitimate FinOps use case deserves its own automation. What should stay consistent is the contribution bar itself: practical implementation, reusable engineering, safe defaults, and a clear connection to real FinOps outcomes.
