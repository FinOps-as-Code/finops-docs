Overview

Shared Documentation for FinOps-as-Code

This repository provides the public documentation layer for the FinOps-as-Code organization, giving projects a shared place for guidance, standards, and future documentation growth.

Purpose

  • Provide a public documentation home through GitHub Pages.
  • Centralize reusable guidance across FinOps-as-Code repositories.
  • Support contributors with shared expectations and good engineering practices.
  • Keep documentation lightweight while the organization grows.

Current Direction

The current organizational focus remains practical FinOps automation, especially executable and independent scripts. Documentation exists to make those contributions easier to understand, adopt, and review.

Why a Separate Docs Repository

  • It creates a stable home for organization-level guidance.
  • It avoids duplicating the same core documentation in every repository.
  • It supports GitHub Pages without coupling the site to a single automation project.
  • It leaves room for future standards, governance notes, and shared examples.

Repository Scope Today

Today this documentation repository is intentionally simple. It includes a public home page, documentation entry points, shared contributing guidance, and baseline standards. Over time it can grow into a more complete docs site if that serves the organization well.
