Overview

What FinOps-as-Code Publishes Today

FinOps-as-Code is building practical automation that helps teams act on cloud cost opportunities, not only describe them. The current repositories focus on lightweight, executable solutions that can be inspected, adapted, and run in real environments.

Portuguese Version

Product Focus

The current product direction is clear: publish FinOps automation that solves real operational cost problems with direct execution paths. The emphasis is on scripts and lightweight workflows that teams can adopt without standing up a platform first.

What The Current Repositories Solve

  • cleanup_block_storage_idle targets unused block storage resources that continue to generate avoidable cloud cost.
  • cleanup_ip_idle focuses on reclaiming idle IP-related resources that create waste or weak governance.
  • linktree-f2-brazil supports the public visibility layer of the organization ecosystem.

What Makes A Good Fit

A strong repository in this organization should solve a concrete FinOps problem, be simple to understand, safe to evaluate, and realistic for teams to adapt in their own environments. The goal is not to publish complexity. The goal is to publish useful automation that turns identified waste or governance gaps into action.

How To Read This Portfolio

Start with the repository that matches the operational problem you want to solve. Then use the contributing and standards sections to understand the engineering bar, safety expectations, and contribution shape expected across the organization.
